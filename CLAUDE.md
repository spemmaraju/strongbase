# Project instructions

## General
- Stay within this project folder unless I explicitly approve otherwise.
- Explain what you’re changing in beginner-friendly language.
- Propose a short plan before major edits.

## Workflow
- Make small, reviewable changes.
- Run the smallest useful verification step after changes.
- Commit at logical checkpoints with clear messages.

## Permissions
- Existing GitHub access is available.
- Before pushing, opening PRs, changing repo settings, or doing anything irreversible, ask first.
- Ask before using web access, external connectors, cloud services, or account access beyond already-approved GitHub access.
- Ask before installing new dependencies or changing toolchain settings.

## Output style
- Keep updates concise.
- Summarize diffs and decisions rather than narrating every tiny action.
